# OpenCLI 常见问题（FAQ）

## 需要登录的 web 页面，这个系统是怎么操作的？

**简要回答**：OpenCLI **不会**替你输入账号密码或托管登录凭证。需要登录的网站，依赖你在 **本机 Chrome 里已经登录**（或通过 **自定义请求头** 等方式提供令牌）。CLI 通过 **Browser Bridge**（扩展 + 本机守护进程）在 **已登录的浏览器会话**里执行页面脚本或带 Cookie 的请求，从而复用你在网页上的登录态。

### 1. 核心思路：复用 Chrome 里的登录态

- 安装并启用 **Browser Bridge** 扩展后，运行需要浏览器的命令时，守护进程会在本机拉起，CLI 与 Chrome 里的扩展通过 WebSocket 通信。
- 扩展在**真实页面上下文**里执行逻辑，能访问该站点在 Chrome 里已有的 **Cookie / 本地存储**，因此等价于「你本人已在该站登录」时的行为。
- 文档说明：使用前请在 Chrome 中打开目标站点并完成登录；会话过期时，在浏览器里重新登录或刷新即可。

### 2. 命令里的「策略」在做什么

适配器会为每条命令声明 **strategy**（在 YAML 或 TS 的 `cli({ ... })` 里），常见与登录相关的是：


| 策略         | 含义（与登录的关系）                                                                 |
| ---------- | -------------------------------------------------------------------------- |
| **public** | 公开接口，不需要登录。                                                                |
| **cookie** | 依赖浏览器会话（Cookie）。执行前往往会 **先导航到站点域名**，让当前标签页处于正确域下，再发请求或执行脚本，以便带上已登录 Cookie。 |
| **header** | 用环境变量等配置的 **Authorization / 自定义头**，适合 API Token 类服务，不依赖你在 Chrome 里点登录。     |


部分复杂站点还会用到 **intercept**（抓 XHR/GraphQL）、**ui**（无障碍/DOM 操作）等，仍建立在「已有会话」或页面交互前提上。

### 3. 你需要做什么

1. 按文档安装 **Browser Bridge** 扩展，并用 `opencli doctor` 检查连接。
2. 在 **Chrome** 中打开目标网站，**手动完成登录**（若尚未登录）。
3. 再运行 `opencli <site> <command> ...`。若出现未授权、空数据，优先检查：是否仍在线、Cookie 是否过期、是否需在浏览器里重新登录。

### 4. 和「Electron 站点」的区别

像 Cursor、Notion 等 **桌面客户端** 不是普通 Chrome 标签页，OpenCLI 会通过 **CDP** 连到对应应用自带的 Chromium 调试端口，同样是复用**该应用内**的登录态，而不是把密码交给 CLI。

### 5. 隐私与安全提示

CLI 只通过本机桥接使用你当前浏览器（或 Electron 应用）里已有的会话，**不在仓库或配置文件里保存你的网站密码**（除非某适配器明确要求你设置环境变量存放 Token，那属于 **header** 类用法，需自行保管）。

## 为什么有些命令要跑十几秒甚至更久？

**常见原因（叠加起来就是「体感很慢」）：**

1. **Browser Bridge 冷启动**
  第一次跑需要拉起守护进程、等 Chrome 扩展连上 WebSocket，会比后续几次慢。
2. **真实页面加载与 SPA**
  很多适配器不是「调一个 JSON API」，而是 **打开真实网页**（或先打开首页再跳进子页）。页面要下载 JS、跑前端框架、拉接口，耗时会直接算进命令里。
3. **UI 自动化里的固定等待**
  为避免白屏、未挂载的编辑器，适配器里常有 `goto` 的 **settle** 时间、`**wait(n)` 秒**，以及「等编辑器出现」「等发布成功提示」等 **轮询超时**（例如最多等 25～30 秒才放弃）。
4. **以 `toutiao weitoutiao` 为例**（`src/clis/toutiao/weittoutiao.ts`）
  为减少「微头条发布页白屏」，会先 `**goto` 创作者首页 `mp.toutiao.com`**（含 settle 与等待），再 `**goto` 微头条发布页**；单这两段导航和等待就很容易到 **10 秒以上**，再加上连浏览器、等编辑器、点发布、等成功提示，**总计十几秒是预期行为**，不是 CLI「卡死」。
5. **网络与站点本身**
  网速慢、站点限流、企业代理拦截脚本等，都会让页面加载或接口变长。

**如何确认时间花在哪？**  
使用 `**opencli -v ...`**（或子命令上的 `-v`）：`[flow]` 行会带**本机时间戳**（`HH:mm:ss.SSS`），可看相邻两行间隔；YAML 管线还会打印各 step 的调试信息。

## `-v` 里每一行 `[flow]` 在说什么？（简约对照）

下面以 `opencli -v toutiao weitoutiao "内容"` 为例；其它站点命令顺序大致相同，个别行会因 **是否走 Electron、是否要预导航、是 TS 还是 YAML** 而略有差异。


| 日志片段                                          | 含义                                                                                                      |
| --------------------------------------------- | ------------------------------------------------------------------------------------------------------- |
| `execute …/…`                                 | 开始执行这条注册命令（`站点/子命令`）。                                                                                   |
| `strategy …`                                  | 该命令声明的策略（如 `public` / `cookie` / `ui` 等），用于文档与部分路由逻辑，不等于「只在这一种方式下跑」。                                    |
| `args …`                                      | 已解析并校验后的参数（你在命令行里传入的内容）。                                                                                |
| `hook onBeforeExecute finished`               | 插件钩子 **执行前** 已跑完（若有插件注册）。                                                                               |
| `browser … required`                          | 本条命令需要 **浏览器会话**（与页面或 Bridge 交互），不是纯本地 HTTP。                                                            |
| `bridge daemon=… extension=…`                 | **Browser Bridge**：本机守护进程是否在跑、扩展是否已连上（仅非 Electron 路径会出现）。若走桌面应用则会看到 **electron / CDP** 相关行。             |
| `env … satisfied`                             | 若命令要求特定环境变量，此处表示已满足。                                                                                    |
| `factory using BrowserBridge`                 | 使用 **浏览器桥** 连接 Chrome（若为 Electron 站点则显示 **CDPBridge**）。                                                 |
| `session open … workspace=…`                  | 正在建立浏览器会话；`workspace` 用于区分不同站点/上下文。                                                                     |
| `… connected — page handle ready`             | 已连上，拿到 `**page`**（后续 `goto`、脚本都在此句之后）。                                                                  |
| `pre-nav …`                                   | **预导航**：部分命令会先打开站点根域名以带上 Cookie。显示 **skipped** 表示该命令在定义里关闭了预导航（如 `navigateBefore: false`），由适配器自己决定怎么跳转。 |
| `timeout … command body`                      | 本条命令**整体**允许运行的最长时间（秒），超时会被中断。                                                                          |
| `handler runCommand`                          | 进入核心分发：准备执行 **TS 函数**或 **YAML 流水线**。                                                                    |
| `adapter dynamic import …`                    | **懒加载**：第一次执行时从磁盘加载对应适配器 `.js` 文件。                                                                      |
| `adapter module registered …`                 | 适配器里的 `cli()` 已执行，命令注册进内存。                                                                              |
| `invoke TypeScript handler` 或 `YAML pipeline` | 真正调用业务逻辑：**函数**或 **流水线步骤**。                                                                             |
| **（时间戳跳跃较大的一段）**                              | **主要耗时在这里**：打开网页、等 SPA、填表、点击、等接口或提示等，都在适配器内部，中间不一定再打 `[flow]`。                                          |
| `session close …`                             | 关闭本次浏览器会话/factory，释放连接。                                                                                 |
| `done finished in …ms`                        | 从进入 `execute` 到结束的总耗时（毫秒）。                                                                              |
| `hook onAfterExecute finished`                | 插件钩子 **执行后** 已跑完。                                                                                       |
